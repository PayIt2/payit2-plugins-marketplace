Plan and set up a new PayIt2 event for: $ARGUMENTS

Follow the event-planning workflow step by step:

1. **Event Details Interview**: Ask the organizer about their event. Cover event type, date/time, location (venue or virtual), capacity, target audience, and what makes this event special or different. Ask them ONE AT A TIME. Wait for each answer before asking the next. Build on previous answers to make follow-ups natural and conversational.

2. **Ticket Tier Design**: Design 2-4 ticket tiers based on the event type and audience. Present them in a table with Tier Name, Price, What's Included, and Quantity Available. If the event date is 4+ weeks out, recommend an early bird tier with 15-20% off that expires 2 weeks before the event. Let the organizer adjust pricing and inclusions.

3. **Event Description Writing**: Write the event page description using the Hook > Details (what/when/where) > Who It's For > What's Included > Logistics framework. Keep it under 200 words. Present it for review and iterate based on feedback.

4. **Visual Strategy**: Ask what photos or graphics the organizer has available. Recommend which to use as the primary event image. If they don't have strong visuals, suggest branded graphics with event name, date, and location overlaid. Recommend dimensions for the main listing image and social share thumbnails.

5. **Promotion Plan**: Generate the complete launch content package:
   - 3 social media posts: announcement ("Save the date"), early bird urgency ("Early bird ends [date]"), and share request ("Know someone who'd love this? Tag them")
   - An email announcement template for the organizer's existing list
   - A text/WhatsApp message for direct outreach to likely attendees
   - A promotion timeline from now to event date with key milestones (announce, early bird close, last chance, day-of reminders)

6. **Attendee Communication Setup**: Draft the registration confirmation email (what they bought, event details, what to bring/expect, calendar link reminder) and a first reminder template to send 3 days before the event.

Save all generated content to a markdown file in the user's workspace folder so they can reference it during event setup and promotion.

Throughout this process, coach the organizer using data: events with early bird pricing sell 40% of tickets before the regular window opens, events promoted across 3+ channels see 2x attendance, and a strong primary image increases click-through by 60%.
