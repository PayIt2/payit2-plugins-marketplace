Set up a PayIt2 group collection for: $ARGUMENTS

Follow the group-collection workflow step by step:

1. **Collection Details Interview**: Ask the organizer about the collection. Cover what it's for, how many people are in the group, total amount needed, whether the per-person amount is fixed or flexible, the deadline for collecting, and what payment options to accept. Ask them ONE AT A TIME. Wait for each answer before asking the next. Build on previous answers to make follow-ups natural and conversational.

2. **Page Setup**: Generate the collection page content. Title should follow the format "[Group] [Purpose] Collection" (e.g., "Office Team Holiday Party Collection"). Write a description that includes the amount breakdown, deadline, and a clear explanation of where the money goes. Present it for review and iterate based on feedback.

3. **Cost Breakdown**: Calculate the per-person amount if fixed. Present a clear breakdown table showing Total Needed, Number of People, Per-Person Cost, and What It Covers. If the collection has tiered options (e.g., "attending dinner" vs "attending dinner + gift"), show each option with its price and what's included. Round up per-person amounts to clean numbers and note any surplus goes toward.

4. **Launch Message**: Generate the initial message to send to the group. Create versions personalized for each channel:
   - Group text / iMessage
   - Email to the group
   - Slack or Teams message
   - Social media post (if applicable)
   Each version should include a payment link placeholder, the per-person amount, the deadline, and a one-line explanation of what it's for. Keep the tone casual and direct — this is friends or colleagues, not strangers.

5. **Reminder Schedule**: Generate 5-6 reminder messages following the "Friendly Collector" cadence:
   - **Launch day**: Initial ask with full context and link
   - **Day 3 progress update**: "X of Y people have paid — we're [percent] there!"
   - **Midpoint momentum**: Celebrate progress, name who's paid if appropriate, gentle nudge
   - **Pre-deadline urgency** (2 days before): "Deadline is [day] — still need X people"
   - **Deadline day**: "Last call — collecting today so we can [purpose]"
   - **Post-deadline follow-up**: Graceful close for stragglers, offer alternative if needed
   Frame every message as a progress update, never as an accusation. Make it easy for people to feel good about paying, not guilty about not paying yet.

6. **Tracking Plan**: Recommend how to communicate progress to the group (e.g., pinned message with a running tally, weekly update in the group chat). Suggest how to handle stragglers — a private 1:1 message template that's friendly and gives them an easy out if money is tight.

Save all generated content to a markdown file in the user's workspace folder so they can reference it throughout the collection.
